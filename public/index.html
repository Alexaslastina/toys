Routes

All HTTP methods are supported. You can use http or https for your requests.

  • https://toysdisneyland.herokuapp.com/toys/ (GET)- First ten toys: /toys/
  • https://toysdisneyland.herokuapp.com/toys/?page=1 (GET)- All toys by pages: /toys/?page=num
  • https://toysdisneyland.herokuapp.com/toys/?s=doll (GET)- Gives an option to search specific toy by name or info: /toys/?s=[name or info]
  • https://toysdisneyland.herokuapp.com/toys/cat/car (GET)- Gives an option to look for toys by category:/toys/cat/catName. There is three options: cars, dolls and lego.

  • In order to add a new toy the client must send (POST) request to main adress https://toysdisneyland.herokuapp.com/toys/ as following example { "name": "cleo", "info": "baby-born", "category": "doll", "img_url": "https://www.pexels.com/ru-ru/photo/208087/", "price": 600 }In order to add a toy, the client has to log in, copy the received authentication token and after that client must include the authentication token in header inside the request.

  • In order to update existing toy the client must send (PUT) request to main adress providing the id of the item you need to update https://toysdisneyland.herokuapp.com/toys/"itemforupdateID" as following with the property changed: example { "name": "house", "info": "constructor", "category": "lego", "img_url": "https://www.pexels.com/ru-ru/photo/208087/", "price": 700 } Authentication token needed in order to fulfill update request. Client must include the authentication token in header inside the request.

  • In order to delite existing toy the client must provide id of the item as follows (DEL) https://toysdisneyland.herokuapp.com/toys/"itemforDELETEID" Authentication token needed in order to fulfill update request. Client must include the authentication token in header inside the request.
